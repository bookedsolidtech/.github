## What We Do

Booked Solid Technology is a consulting practice and product studio. We architect systems, build platforms, ship AI integrations, and lead engineering teams — all at principal level, all in production.

**Consulting** — Architecture, AI/LLM strategy, fractional CTO, engineering leadership, design systems, code audit. The engineers you talk to are the engineers doing the work. No handoffs. No juniors.

**Platform Development** — Full-stack product builds from zero to production. SaaS platforms, booking engines, AI-powered analysis tools, real-time collaborative apps. We own the outcome end-to-end.

**AI & Autonomous Systems** — We don't just advise on AI, we build with it. RAG pipelines, agentic workflows, MCP tooling, autonomous development infrastructure. Our own development process runs on AI agents we built ourselves — that's not a pitch, it's how this code got here.

**Open Source** — The tools we build for ourselves, we release for everyone. Enterprise web components, AI developer tooling, and the infrastructure that ties it all together.

## Open Source Projects

### [HELiX](https://github.com/bookedsolidtech/helix) — Enterprise Web Component Library

Production-grade Web Components built with Lit 3.x and TypeScript. 40+ components, W3C DTCG design tokens, WCAG 2.1 AA accessible. Framework-agnostic — works with React, Astro, Vue, Angular, Drupal, or plain HTML.

[![Build](https://img.shields.io/github/actions/workflow/status/bookedsolidtech/helix/ci.yml?branch=main&label=CI&logo=github)](https://github.com/bookedsolidtech/helix/actions)
[![TypeScript](https://img.shields.io/badge/TypeScript-strict-3178C6?logo=typescript&logoColor=white)](https://github.com/bookedsolidtech/helix)
[![Lit](https://img.shields.io/badge/Lit-3.x-324FFF?logo=lit&logoColor=white)](https://lit.dev)
[![WCAG 2.1 AA](https://img.shields.io/badge/WCAG%202.1-AA-success)](https://github.com/bookedsolidtech/helix)

- `@helix/library` — Buttons, cards, grids, navigation, forms, structured lists, and more
- `@helix/tokens` — Design tokens with light/dark/high-contrast themes
- Storybook 10.x + Astro/Starlight documentation
- Built for healthcare and enterprise — where accessibility isn't optional

### [HELiXiR](https://github.com/bookedsolidtech/helixir) — AI-Powered Component Intelligence

MCP server that gives AI coding agents accurate awareness of any web component library. Reads your Custom Elements Manifest and design tokens so AI stops hallucinating component APIs.

[![npm](https://img.shields.io/npm/v/helixir?logo=npm)](https://www.npmjs.com/package/helixir)
[![Downloads](https://img.shields.io/npm/dw/helixir)](https://www.npmjs.com/package/helixir)
[![Build](https://img.shields.io/github/actions/workflow/status/bookedsolidtech/helixir/build.yml?branch=main&label=build&logo=github)](https://github.com/bookedsolidtech/helixir/actions)

- Works with any component library — Shoelace, Spectrum, FAST, HELiX, yours
- Properties, events, slots, CSS parts, design tokens — all from source of truth
- Plug into Claude, Cursor, Windsurf, or any MCP-compatible tool

```bash
npx helixir init && npx helixir
```

## How We Build

We practice what we ship. Our own development runs on autonomous AI agents, continuous integration, and the same engineering standards we bring to client work:

- **Strict TypeScript everywhere** — no `any`, no shortcuts
- **Automated quality gates** — CI, linting, formatting, accessibility audits
- **AI-augmented development** — agentic pipelines, autonomous PR creation, AI code review
- **Web standards first** — Web Components, W3C DTCG tokens, MCP protocol, semantic HTML
- **Production or it doesn't ship** — every deliverable held to the standard of code we'd deploy ourselves
